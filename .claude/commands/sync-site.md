The landing site (`site/src/components/*.astro`) **hardcodes** numbers and claims that are *lifted from*
`docs/` — it does not derive them. When a doc is edited, the site copy silently drifts, and CI only greps
that four headline strings *exist*, never that they still *match* the source. Your job: find and close that
drift. `docs/` is the source of truth; the site is downstream.

## Scope

**$ARGUMENTS**

- empty → **full audit** of every coupling in the map below.
- a doc path (e.g. `docs/proven.md`) → audit only the components fed by that doc.
- `since <git-ref>` (e.g. `since HEAD~5`) → `git diff <ref>..HEAD -- docs/` first, then audit only the
  components fed by the docs that changed.

Start by establishing what moved:

```bash
git log --oneline -10 -- docs/
git diff HEAD~5..HEAD -- docs/    # widen the range as needed to cover the edits in question
```

export-run`, not a doc. If a run number looks stale, say so and stop — don't hand-edit it.

## Method

1. For each in-scope doc, read it, then read the component(s) it feeds. Compare **number by number and
   claim by claim** — the site rewords the docs, so match on *meaning*, not on string equality.
2. Build a drift table before touching anything:

   | source (doc:line) | doc says | site says (file:line) | verdict |
   |---|---|---|---|
   | proven.md:9 | redundant work fell 53% (17 → 8) | `−53%` / `17 → 8` (Proof.astro:18) | ✓ match |

   Verdicts: `✓ match` · `✗ drift` (number/claim changed) · `⚠ orphan` (site claims something the doc no
   longer supports) · `＋ missing` (a doc change the site should reflect but doesn't).
3. Fix only `✗ / ⚠ / ＋` rows, editing the `.astro` files. **Preserve the site's terser voice** — reword to
   fit, don't paste doc prose. **Never invent a number**: every figure on the site must trace to a doc line.
   If a doc *removed* a claim, remove or flag the site copy — do not fabricate a replacement.
4. If a headline number in the CI grep changes (`99.9%`, `2.4ms`, `93.3%`, `−53%` in
   `.github/workflows/site.yml`), update that grep list to match so CI stays honest.

## Verify (matches the `site` CI job)

```bash
cd site
pnpm check-tokens          # design tokens still a verbatim copy of the dashboard
pnpm check                 # astro type/content check
pnpm build                 # env not needed for a drift check
grep -q "99.9%" dist/index.html && grep -q "2.4ms" dist/index.html \
  && grep -q "93.3%" dist/index.html && grep -q "−53%" dist/index.html
node scripts/check-budget.mjs
```

## Report

Print the drift table, then a short changelog of the edits (which claim, doc→old→new, file:line). If every
row is `✓ match`, say so plainly — "site in sync with docs, no edits" — and make no changes. Do not commit.
